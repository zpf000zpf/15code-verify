# 15code Verify — 榜单政策（Leaderboard Policy）

**版本：v1.0 · 生效日期：2026-04-23**

---

## 一、核心立场

15code Verify 是**技术工具 + 数据聚合平台**，**不是评级机构**。本榜单：

- 不对任何服务商作出商业评价或法律判断
- 仅呈现由社区用户自愿匿名贡献的技术观测数据的统计结果
- 所有数据均基于公开可复现的方法论

## 二、数据来源

榜单数据来源**仅限于**：

1. 用户使用 15code Verify 扫描后**明确勾选 ✅ "允许匿名贡献到榜单"** 的记录
2. 15code 官方基线扫描（每周对各官方 API 执行，作对照基准）

以下数据**不会**进入榜单：
- 用户未授权的扫描数据
- 用户个人 IP / 时间 / 身份信息
- prompt 原文 / response 原文

## 三、展示规则

### 3.1 样本门槛
- 单一供应商**少于 100 个扫描样本**不公开展示
- 最近 30 天样本少于 30 个，条目标"数据过期"

### 3.2 呈现形式
- 所有指标以**均值 ± 置信区间**展示
- 永远不展示单次扫描记录
- 永远不使用"造假"、"欺诈"、"冒充"等主观措辞

### 3.3 技术中立措辞
| ❌ 不使用 | ✅ 使用 |
|---|---|
| "A 商家造假" | "响应模式与官方 Claude Opus 一致性：12% ± 3%" |
| "偷换模型" | "能力差分得分：与 claimed model 基线差 X 个标准差" |
| "骗子" | "（无任何主观评价，仅数据）" |

## 四、供应商权益

### 4.1 申辩权（Right of Reply）
每个榜单条目设有"供应商回复"入口。供应商可提交：
- 官方说明 / 声明
- 自己的检测报告
- 对特定探针的技术质疑

回复内容由供应商自行负责，与"用户数据"并列展示。

### 4.2 复查申诉（Dispute）
- 邮箱：dispute@15code.com
- 48h 内响应确认
- 技术复查期间条目标"审查中"
- 确认错误后 24h 内更新数据 + 公示更正

### 4.3 退出机制
- 供应商可申请"**不公开**"（但不能修改数据）
- 退出后条目标"应要求不展示"
- 原始数据仍保留 12 个月以备监管审查

## 五、方法论透明

- 打分算法 / 探针设计原理**完全开源**，版本化发布
- 详见：[PROBE_DESIGN.md](PROBE_DESIGN.md)
- 任何人可以复现或提出质疑

## 六、免责声明

> 本榜单数据由社区用户使用 15code Verify 开源工具扫描后**自愿匿名贡献**汇总得出。  
> 数据反映**特定时间点的技术指标观测**，**不构成**：
> - 对任何服务商商业行为的法律评价
> - 对任何供应商商品质量的官方结论
> - 交易决策或投资建议
>
> 用户应自行判断并承担决策后果。

## 七、发布阶段

| 阶段 | 启动条件 | 榜单形态 |
|---|---|---|
| Phase 1 | 工具上线 | **不发布榜单**，仅个人私有报告 |
| Phase 2 | 累计扫描 ≥ 10,000 次 | **白名单模式**：只展示愿意被展示的供应商（15code 自己必在） |
| Phase 3 | 单供应商样本 ≥ 500 | 全量聚合榜 + 申辩机制 |
| Phase 4 | 数据稳定 6 个月+ | 增加趋势图、历史追溯 |

## 八、更新机制

本政策可能随法律法规、社区反馈更新：
- 重大变更提前 30 天公告
- 历史版本可在 [GitHub](https://github.com/zpf000zpf/15code-verify) 追溯
- 欢迎通过 Issue / Discussion 反馈
